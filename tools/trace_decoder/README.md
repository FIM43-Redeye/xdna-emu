# trace_decoder

Independent re-implementation of the AIE2 on-tile trace decoder.

The runtime trace pipeline used to depend on mlir-aie's `parse_trace`
(open-source, fine) and was about to start depending on aietools'
shared libraries (closed-source, ambiguous license -- not OK).  This
package replaces both with an MIT-licensed decoder we own end-to-end.

## Status

| Mode | Name        | Status            |
|------|-------------|-------------------|
| 0    | EVENT_TIME  | bit-equivalent with mlir-aie on fixtures |
| 1    | EVENT_PC    | TODO -- format reverse-engineering active |
| 2    | INST_EXEC   | TODO -- branches + ZOL LC, cardano frame taxonomy |
| 3    | reserved    | TODO -- empirically produces 896-byte payloads |

## Reference sources used (read-only -- no code or data copied)

* `mlir-aie/python/utils/trace/{parse,utils}.py` (Apache 2.0) -- mode-0
  byte format is fully specified there.
* `aietools/include/drivers/aiengine/xaiengine/xaie_trace.h` (MIT) --
  `XAie_TraceMode` enum.
* `aietools/include/adf/adf_api/BaseImpl.h` (MIT) -- `event_trace_mode`
  enum (alternative public surface for the same values).
* `aietools/lib/lnx64.o/libevent_trace_decoder.so` -- read-only symbol
  inspection of `adf::Trace::TraceDecoder`.  Library is never linked or
  shipped with this code.
* `aietools/lib/lnx64.o/libxv_trace_decoder_opt.so` -- read-only symbol
  inspection of `cardano::Trace::*` for the mode-2 frame taxonomy.

See `xdna-emu/docs/observability-leads.md` for the full reverse-
engineering notes.

## Layout

```
trace_decoder/
  __init__.py        Public API
  frame.py           TraceCommand / Event / TraceMode / PacketType schema
  packet.py          32-byte stream-switch packet header + de-interleave
  decode.py          Top-level orchestration
  modes/
    mode0.py         EVENT_TIME byte-stream decoder
    mode1.py         (TODO) EVENT_PC byte-stream decoder
    mode2.py         (TODO) INST_EXEC byte-stream decoder
  fixtures/
    mode0_*.bin      Raw trace captures
    mode0_*.expected.json  mlir-aie oracle output (regenerated by
                           regenerate-fixtures.sh)
```

## Tests

`tools/test_trace_decoder.py`.  Run with:

```
./scripts/run-tests.sh --lib   # not needed; this is a Python test
pytest tools/test_trace_decoder.py -v
```

The module-level test `test_mode0_decode_matches_oracle_byte_for_byte`
is the hard correctness contract: per-tile command sequences must
match mlir-aie's `convert_to_commands` after canonicalization (we
collapse Single0/1/2 size variants into one shape, since the size
distinction is an encoder-side compression detail rather than a
semantic one).

## Regenerating fixtures

```
tools/trace_decoder/regenerate-fixtures.sh
```

This runs mlir-aie's `convert_to_commands` over the `.bin` captures
and dumps the result to `*.expected.json`.  Only re-run when you
deliberately want to refresh the oracle (e.g. after adding a new
fixture).
