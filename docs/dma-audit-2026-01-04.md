## 1. Buffer Descriptor (BD) Format

### Documentation Specification (AM025 memory_module/dma/bd.txt)

Each BD consists of **6 words (24 bytes)**, with 32-byte register stride:

| Word | Bits | Field | Purpose |
|------|------|-------|---------|
| 0 | 27:14 | Base_Address | Data memory address (word units, 14 bits) |
| 0 | 13:0 | Buffer_Length | Transfer size in 32-bit words (14 bits) |
| 1 | 31 | Enable_Compression | MM2S compression enable |
| 1 | 30 | Enable_Packet | MM2S add packet header |
| 1 | 29:24 | Out_Of_Order_BD_ID | Packet header OOO ID field |
| 1 | 23:19 | Packet_ID | Packet header ID field |
| 1 | 18:16 | Packet_Type | Packet header type (3 bits) |
| 2 | 25:13 | D1_Stepsize | Dimension 1 stride (actual-1, words) |
| 2 | 12:0 | D0_Stepsize | Dimension 0 stride (actual-1, words) |
| 3 | 28:21 | D1_Wrap | Wrap count D1 (0=no wrap) |
| 3 | 20:13 | D0_Wrap | Wrap count D0 (0=no wrap) |
| 3 | 12:0 | D2_Stepsize | Dimension 2 stride (actual-1, words) |
| 4 | 24:19 | Iteration_Current | Current iteration step |
| 4 | 18:13 | Iteration_Wrap | Iteration wrap count (actual-1) |
| 4 | 12:0 | Iteration_Stepsize | Per-iteration offset (actual-1, words) |
| 5 | 31 | TLAST_Suppress | MM2S only: suppress TLAST at end |
| 5 | 30:27 | Next_BD | Next BD index (4 bits) |
| 5 | 26 | Use_Next_BD | Enable chaining |
| 5 | 25 | Valid_BD | BD valid/enabled |
| 5 | 24:18 | Lock_Rel_Value | Lock release signed value (7 bits) |
| 5 | 16:13 | Lock_Rel_ID | Lock release ID (4 bits) |
| 5 | 12 | Lock_Acq_Enable | Lock acquire enable |
| 5 | 11:5 | Lock_Acq_Value | Lock acquire signed value (7 bits) |
| 5 | 3:0 | Lock_Acq_ID | Lock acquire ID (4 bits) |

## 2. Lock Integration

### Documentation Specification

**Lock Model** (AM025, AM020 Ch2):
- 16 locks per compute tile (4-bit IDs: 0-15)
- 64 locks per memory tile (6-bit IDs: 0-63)
- 6-bit unsigned semaphore value (0-63)
- No "acquired" bit - pure counting semaphore

**Acquire Semantics** (based on Lock_Acq_Value sign bit):
- **Negative value (bit 6 = 1)**: `acquire_greater_equal` - wait until lock >= |value|
- **Non-negative value (bit 6 = 0)**: `acquire_equal` - wait until lock == value

**Release Semantics**:
- Add Lock_Rel_Value to lock
- Saturate at 0 (floor) and 63 (ceiling)

### Current Implementation Issues

**File**: `src/device/tile.rs` lines 98-170

**Issue 1: No acquire_ge vs acquire_eq distinction**
```rust
// CURRENT CODE - doesn't check sign bit
pub fn try_acquire(&mut self, expected: u8) -> LockResult {
    if self.value >= expected {  // Always uses >= semantics!
        // ...
    }
}
```

**Correct Implementation**:
```rust
pub fn try_acquire(&mut self, value: i8) -> LockResult {
    let is_ge_mode = value < 0;  // Negative = acquire_ge
    let threshold = value.abs() as u8;

    if is_ge_mode {
        // acquire_greater_equal: wait for lock >= threshold
        if self.value >= threshold {
            self.value = self.value.saturating_sub(threshold);
            LockResult::Acquired
        } else {
            LockResult::WouldBlock
        }
    } else {
        // acquire_equal: wait for lock == threshold
        if self.value == threshold {
            self.value = 0;  // Reset to 0 on acquire_eq
            LockResult::Acquired
        } else {
            LockResult::WouldBlock
        }
    }
}
```

**Issue 2: Release only handles +1**
```rust
// CURRENT CODE
pub fn release(&mut self) {
    if self.value < Self::MAX_VALUE {
        self.value += 1;  // Only +1!
    }
}
```

**Correct Implementation**:
```rust
pub fn release(&mut self, delta: i8) {
    let new_value = (self.value as i16) + (delta as i16);
    self.value = new_value.clamp(0, 63) as u8;
}
```

**Issue 3: Locks never enforced in DMA engine**

**File**: `src/device/dma/engine.rs`

The engine parses lock fields but never actually:
1. Waits on lock acquire before starting transfer
2. Releases lock after transfer completes

**Expected Transfer Lifecycle**:
```
1. Transfer created from BD
2. If Lock_Acq_Enable:
   a. Attempt lock acquire
   b. If blocked, set state = WaitingForLock
   c. Return, will retry next step()
3. Once lock acquired (or not needed), proceed with data transfer
4. On transfer complete:
   a. If Lock_Rel_Value != 0, release lock
5. Mark transfer complete
```

## 3. Channel Control Registers

### MM2S Channels (AM025 memory_module/dma/mm2s.txt)

**Control Register** (offset 0x1DE10 + ch*8):
- Bit 4: Compression_Enable (master enable)
- Bits 15:8: Controller_ID (task-complete-token ID)
- Bit 1: Reset
- Bit 0: Reserved

**Start Queue Register** (offset 0x1DE14 + ch*8, write-only):
- Bit 31: Enable_Token_Issue
- Bits 23:16: Repeat_Count
- Bits 3:0: Start_BD_ID

**Status Register** (offset 0x1DF10 + ch*4):
- Bits 27:24: Cur_BD
- Bits 22:20: Task_Queue_Size
- Bit 19: Channel_Running
- Bit 18: Task_Queue_Overflow
- Bit 11: Error_BD_Invalid
- Bit 5: Stalled_TCT
- Bit 4: Stalled_Stream_Backpressure
- Bit 3: Stalled_Lock_Rel
- Bit 2: Stalled_Lock_Acq
- Bits 1:0: Status (00=IDLE, 01=STARTING, 10=RUNNING)

### S2MM Channels (AM025 memory_module/dma/s2mm.txt)

**Control Register** (offset 0x1DE00 + ch*8):
- Bits 17:16: FoT_Mode (Finish-on-TLAST)
  - 00: Disabled
  - 01: FoT_no_counts
  - 10: FoT_counts_with_task_tokens
  - 11: FoT_counts_from_mm_register
- Bit 4: Decompression_Enable
- Bit 3: Enable_Out_of_Order

## 4. TLAST and Packet Handling

### TLAST Semantics

**Documentation**: TLAST marks the last 32-bit word of a stream transaction.

**BD Control**:
- Word 5, Bit 31: TLAST_Suppress
- If set, don't assert TLAST at end of transfer
- Used for multi-BD transfers that should appear as single stream

**Current Issue** (`src/device/dma/engine.rs` ~line 515):
```rust
// CURRENT - always sets TLAST on last word
tlast: is_last && i == word_count - 1,
```

**Correct**:
```rust
tlast: if bd.tlast_suppress { false } else { is_last && i == word_count - 1 },
```

### Packet Header Format

When Enable_Packet = 1 (BD Word 1, bit 30), MM2S inserts 32-bit header before data:

| Bits | Field |
|------|-------|
| 31 | Odd parity |
| 30:28 | 3'b000 |
| 27:25 | Source Column |
| 24:22 | Source Row |
| 21 | 1'b0 |
| 20:18 | Packet_Type (from BD) |
| 17:11 | 7'b0000000 |
| 10:5 | Stream_ID (from BD Packet_ID) |
| 4:0 | RESERVED |

**Current Status**: Not implemented - Enable_Packet bit never read.

## 5. Multi-Dimensional Addressing

### Address Generation Formula

```
address = base_address
        + (d0_counter * d0_stepsize)
        + (d1_counter * d1_stepsize)
        + (d2_counter * d2_stepsize)
        + (iteration_counter * iteration_stepsize)
```

### Wrap Behavior

- D0 counter increments each word
- When D0 counter reaches D0_Wrap, reset D0, increment D1
- When D1 counter reaches D1_Wrap, reset D1, increment D2
- When iteration counter reaches Iteration_Wrap, BD complete
